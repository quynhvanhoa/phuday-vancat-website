🚀 Xuất bản nội dung lên website

Sau khi sửa xong nội dung, làm 3 bước sau để website thật hiển thị nội dung mới:

  1. Click "Xuất bản" bên dưới → tải file content.json về máy
  2. Mở GitHub repository của chị → kéo thả file vừa tải vào (ghi đè file cũ) → commit
  3. Đợi 1–2 phút → GitHub Pages tự build → website cập nhật nội dung mới

Phạm vi quản lý của admin (bản demo)

Quan trọng: Phiên bản demo này chỉ quản 4 loại nội dung dưới đây. Khi triển khai thật trên WordPress sẽ quản được TẤT CẢ nội dung trên website (kể cả lời chào, các thẻ giới thiệu, trang Tín ngưỡng, Tour 360...).

✅ Có thể sửa trong demo:

Mục admin Hiển thị trên website
📝 Bài viết Phần "Tin tức nhà đền" trên trang chủ + toàn bộ trang Tin tức
📅 Sự kiện Phần "Lịch tiệc Mẫu trong năm" trên trang Lễ hội
🖼️ Thư viện ảnh Trang Thư viện ảnh (phân theo 4 danh mục)
⚙️ Cài đặt Email, SĐT — hiển thị ở trang Liên hệ và footer

❌ Chưa sửa được trong demo (sẽ có khi triển khai WordPress):

  • Lời chào của thủ nhang trên trang chủ
  • Các thẻ giới thiệu (Tour 360, Tín ngưỡng, Lễ hội, Cung tiến)
  • Nội dung trang Giới thiệu, Tín ngưỡng thờ Mẫu
  • Mô tả các điểm trong Tour 360
  • Bản đồ tour 360 (sẽ là iframe Kuula/3DVista)
  • Logo, banner, ảnh hero

Lý do: Phiên bản demo chạy bằng HTML/JS thuần để chị trải nghiệm nhanh. Một CMS thật (như WordPress) cần backend đầy đủ để quản trị mọi nội dung.

Cách kiểm tra demo hoạt động

  1. Vào 📝 Bài viết → click "Sửa" trên 1 bài → đổi tiêu đề → bấm Lưu
  2. Mở Trang chủ (mở tab mới) → cuộn xuống mục "Tin tức nhà đền" → thấy tiêu đề mới
  3. Nếu KHÔNG thấy thay đổi: nhấn Cmd+Shift+R để tải lại không cache
  4. Lưu ý: phải mở qua http://localhost:8765 chứ không phải file://... (Safari chặn localStorage cho file://)
